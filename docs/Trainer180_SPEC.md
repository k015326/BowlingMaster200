3. 入力仕様
   3-1. カスタムキーボード
   残りピン入力時は、以下のキーを持つカスタムキーボードを表示する。

キー	意味	備考
1〜9	残りピン番号	複数入力可（例：36 = 3番と6番残り）
X	ストライク	1投目専用　※アンバー背景で強調表示
/	スペア（全ピン倒）	2投目・10F2投目・10F3投目で使用。isSpareSuccess=trueをセット
G	ガター	スコア計算上0本扱い
F	ファウル	スコア計算上0本扱い・ファウルフラグを別途記録
DEL	1文字削除

3-2. スパット通過精度入力
1投目・2投目の結果入力画面に組み込む。プルダウン方式で、スクロールなしに全11段階を表示。

値	意味
-5〜-1	右にズレた（右から数えて板目が小さい方向）
0	ぴったり通過
+1〜+5	左にズレた（右から数えて板目が大きい方向）

3-3. オイルパターン切り替え
ゲーム入力画面の上部に常時表示するトグルボタン。セッション開始時はハウスがデフォルト。

選択肢	説明	初期アジャスト補正
ドライ	オイルが少ない・枯れたレーン	基準値より外目（板目の数字を小さく）に補正
ハウス	一般的なハウスオイルパターン	補正なし（基準値そのまま）
セミスポーツ	ハウスより難しい・オイル量多め	基準値より内目（板目の数字を大きく）に補正

3-4. ゲーム間休憩メモ
「新しいゲーム開始」ボタン押下前に任意入力。プリセット複数選択＋アジャスト推奨値の組み合わせ。

プリセット例	デフォルトアジャスト推奨値
右レーンが速い	-2
左レーンが遅い	+3
オイルが枯れてきた	-1
疲れてきた	0
調子が良い	0
4. ロジック仕様
   4-1. 立ち位置・スパットの板目表記
   板目は右から何枚目で統一。右投げのため、アジャスト方向は以下の通り。

状況	アジャスト方向
右ピン残り（6・10等）	内（左）方向＝板目の数字を大きく
左ピン残り（7等）	外（右）方向＝板目の数字を小さく
ボールが内に入りすぎ（ブルックリン方向）	内方向＝板目の数字を大きく
ボールが外に出た（ガター方向）	外方向＝板目の数字を小さく

4-2. スペアボール判定ルール
残りピンの先頭ピン（最も投球者側に近いピン）で判定。プロファイルで変更可能。

先頭ピン	使用ボール	デフォルト立ち位置/スパット
10	スペアボール（弱いナチュラルフック）	30/15
6	スペアボール（弱いナチュラルフック）	27/15
3・9	スペアボール（弱いナチュラルフック）	24/15
1・5	ストライクボール（ナチュラルフック）	20/10
2・8	ストライクボール（ナチュラルフック）	17/10
4	ストライクボール（ナチュラルフック）	14/10
7	ストライクボール（ナチュラルフック）	11/10

4-3. 初動3フレームナビ
1〜3フレーム目は基準値（プロファイル設定値）を推奨として表示。
4フレーム目以降は同一セッション内の類似データ（立ち位置±2枚）からストライク率を算出して推奨値を更新。

4-4. 信頼度表示

データ数	信頼度表示	推奨値の扱い
0〜2件	★☆☆（低）	基準値を使用
3〜9件	★★☆（中）	参考値として提示
10件以上かつばらつき小	★★★（高）	実績値として提示

4-5. スコア計算
標準ボウリングルール準拠。ガター（G）・ファウル（F）は0本として計算。
●	ストライク＝10点＋次の2投のピン数
●	スペア＝10点＋次の1投のピン数
●	10フレームは最大3投（thirdPosition / thirdTarget / thirdPins フィールドで管理）
●	10フレーム2投目のスペア（/）も有効

4-6. 10ピン専用アラート
10ピンのスペア失敗が3回連続した場合に発動。
●	画面全体が赤くフラッシュ（0.5秒）
●	ピンが砕けるような効果音（設定ページでON/OFF切り替え可）
●	中央に大きく「⚠ 10ピン連続ミス！」のアニメーションテキスト
●	具体的な立ち位置・スパット調整アドバイスを表示
●	「OK」タップで閉じる

4-7. 目標進捗バー
直近10ゲームの移動平均AVEを現在地として表示。プロファイルで設定した目標AVEと期限を基に進捗を可視化。

4-A. 本日の注意ポイント表示機能

★ 変更点①：ゲーム開始時に過去データを分析して注意ポイントを1点提示
2ゲーム目以降のゲーム開始時に、当セッション内の直近データを分析して「最も改善が必要なポイント」を1つ表示する。
プレイヤーが次のゲームで意識すべき具体的な行動指針を提供することで、上達を加速させる。

■ 表示タイミング
●	1ゲーム目開始時：表示なし（データ不足のため空欄）
●	2ゲーム目以降の開始時：「本日の注意ポイント」カードを表示
●	表示場所：ゲーム入力画面の最上部（オイルパターントグルの直下）

■ 分析ロジック（優先順位順）

優先度	判定条件	表示メッセージ例
1	10ピン成功率 < 50%	「10ピンを確実に取りにいこう。スペアボールで30板目に立ち、15枚目のスパットを通すライン意識で。」
2	スパット通過精度の平均ズレ ≧ ±2	「スパット通過が右に2枚ずれる傾向あり。スパット通過に集中し、板目を1〜2枚左に調整してみよう。」
3	スペア率 < 60%（10ピン除く）	「スペアの精度が低下中。2投目の立ち位置を前ゲームより1枚内側に調整してみよう。」
4	後半フレーム（8〜10F）のストライク率 < 前半の70%	「後半フレームで集中力が落ちる傾向あり。8Fから1投ずつ丁寧に。」
5	直前ゲームのAVE < セッション平均 - 10	「前ゲームはスコアが下がった。力みを抜いてスムーズなスイングを意識しよう。」

■ UIデザイン仕様
●	カード背景：アンバー（#F59E0B）の10%透過＋左ボーダー4dpをアンバーで強調
●	アイコン：⚡（Lightning Bolt）アイコンをアンバーで左端に表示
●	タイトルテキスト：「本日の注意ポイント」（14sp Medium、アンバー）
●	本文テキスト：16sp Regular、#E2E8F0（シルバー）
●	「前のゲームから学ぶ」副題（12sp、グレー）でデータ根拠を明示
●	カードは折りたたみ可能（デフォルト展開）。タップで最小化できる

■ データソース
●	当セッション内の全Frameデータを参照（セッションをまたいだ参照は行わない）
●	GameエンティティのgameNumber（セッション内のゲーム順序）で判定
●	分析はゲーム終了時にバックグラウンドで自動実行し、次ゲーム開始まで保持

■ 実装クラス
●	GameAdviceAnalyzer（UseCase）：セッション内データを受け取り、Advice文字列を返す
●	GameAdviceState：advice: String?、category: AdviceCategory（SPARE_10PIN / ACCURACY / SPARE / STAMINA / RECOVERY）
●	ゲーム入力画面のViewModelで GameAdviceState を StateFlow として保持
