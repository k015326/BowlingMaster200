________________________________________
1. アプリ概要
   1.1 アプリ名
   スペア マスター
________________________________________
1.2 目的
ボウリングのスコアシート画像から、スペア挑戦時の残ピン情報を蓄積し、苦手パターンを可視化する。
本アプリの本質は、
「OCRアプリ」ではなく
「残ピン統計アプリ」
である。
OCRは入力補助機能と位置付ける。
________________________________________
1.3 開発方針（最重要）
基本思想
•	小さく作る
•	まず完成させる
•	一括実装禁止
•	MVP（最小実用版）優先
•	OCRは後回し
•	AIクレジット浪費を避ける
•	無料枠主体でも継続開発できる構成にする
________________________________________
1.4 AI活用方針
現状
•	Cursor Pro 使用中
•	残クレジット約20%
今後
•	Pro終了後は無料枠中心へ移行
•	ChatGPT / Cursor / Claude の無料利用を前提に継続開発

13. SpareAttempt生成条件
________________________________________
記録対象
以下を満たす場合のみ生成。
•	1投目で全倒しではない
•	1番ピン単独残りではない
•	スペア挑戦である
________________________________________
10フレーム
•	1投目・2投目は対象
•	3投目は除外
________________________________________
例
10フレームで7番残り→スペア成功
→ 1件記録

31. 新チャット用 引継ぎプロンプト（重要）
    Android Studio初心者です。

現在、「スペア マスター」というAndroidアプリを開発しています。

■ 開発方針

・小さく段階実装
・一括実装禁止
・1StepごとBuild確認
・未使用コード禁止
・将来用コード先回り禁止
・まず動く最小構成優先
・Cursor無料枠節約重視

■ 技術構成

・Kotlin
・Jetpack Compose
・Room
・MVVM
・CameraX
・ML Kit（将来）

■ 現在の仕様

・富津スターレーン専用
・OCRは補助
・手入力MVP優先
・10bit残ピン管理
・統計アプリが本体

■ 実装順

Step1 PinBitUtils
Step2 ピンUI
Step3 手入力
Step4 Room DB
Step5 統計
Step6 カメラ
Step7 OCR

■ AIへの要求

・今やるStepだけ実装
・仕様外機能追加禁止
・未使用コード禁止
・小さく安全に進める
・Buildが通る最小コードのみ

まずは Step1 を開始してください。
